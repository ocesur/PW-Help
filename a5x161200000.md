#### Elektronik Posta Gönderim Hataları

:::::: {.section .errorBox}
::: title
Bilgi
:::

:::: content
::: {style="text-align: justify;"}
Elektronik posta gönderimi PaperWork mimarisinde standartdır ve global
yönergeler kullanılır. Yapılan tanım ve ayarlara göre zaman zaman
elektronik posta gönderiminde hatalar alınır. SMTP sunucusundan gelen
hataların tamamı için ilgili sunucu kılavuzları incelenmeli veya google
üzerinden aranarak oluşan sorun nedeni tespit edilmelidir. Aşağıda
alınan hatalara örnekler verilmiştir.
:::
::::
::::::

- The operation has timed out. (Sunucu timeout süresi içerisinde posta
  gönderilememiştir. Gönderilecek posta boyutu, SMTP sunucusu yoğunluğu
  gibi durumlarda oluşur) Tekrar gönderilebilir.
- Failure sending mail. 
- The SMTP server requires a secure connection or the client was not
  authenticated. The server response was: 5.7.57 SMTP; Client was not
  authenticated to send anonymous mail during MAIL FROM
  \[AM5PR0602CA0024.eurprd06.prod.outlook.com\]
- A recipient must be specified. (Bu hata gönderim esnasında geçerli bir
  posta adresi bulunmadığını ifade eder)
- The specified string is not in the form required for an e-mail
  address.
- Exceeded storage allocation. The server response was: 5.3.4 Message
  size exceeds fixed maximum message size
  \[AM4P190MB0178.EURP190.PROD.OUTLOOK.COM\] (Bu hata gönderime izin
  verilen boyuttan büyük veri, eklenti gönderildiğini ifade eder)
- The specified string is not in the form required for an e-mail
  address. Gönderim esnasında gönderilecek kullanıcının mail adresi
  doğru formatta değildir veya SMTP sunucusu bu adresi kabul etmiyordur.
- [Elektronik Posta gönderildi olarak raporlara yansımış fakat
  gitmemiş]{.underline}: SMTP sunucusu bu postayı SPAM olarak algılamış
  olabilir. Konunun SMTP sunucusu tarihçelerinden araştırılması gerekir.
- The remote certificate is invalid according to validation procedure.
  Bu hata birden fazla durumda alınabilir. Bunlardan biri de sunucu
  sertifikasının Content Server\'ın olduğu makinede olmaması(geçersiz
  olma) durumunda oluşur. Web ara yüzü ile Content Server üzerinden
  ilgili servis sağlayıcıya (örneğin O365) bağlanılır ise sertifikanın
  son hali istemci makinesine indirileceğinden dolayı sorun
  çözümlenebilir. Çözülemediği durumda internet üzerinden konu
  araştırılaqrak incelenebilir.
- Authentication failed because the remote party closed transport
  stream. SMTP sunucusu SSL sertifikası geçersiz veya yok. Bu durumda
  kodun çalıştığı sunucu üzerinde SMTP web adresine aynı hesap ile bir
  sefer girmek yeterlidir. Bu durumda sertifika istemci makineye de
  otomatik. kurulur.
- *Transaction failed. The server response was: 5.2.252
  SendAsDenied; *Eğer elektronik posta hesabı \"Relay\" gönderimine
  kapalı ise ve belgeler ekranında bir belge gönderilmeye çalışıyor ise
  bu hata alınır. Ya hesabın relay özelliğinin açılması ya da
  tanımlarında \"Relay Aktif\" seçeneği kaldırılmalıdır.

Yukarıda görülen hataların tamamı SMTP sunucusundan gelmiştir ve
PaperWork mimarisi ile alakalı değildir.
