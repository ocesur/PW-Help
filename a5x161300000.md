#### PaperWork Veri Tabanı

PaperWork MS SQL Server 2016 Standart 64 bit ve üzeri versiyonlarında
sağlıklı bir şekilde çalışır. Geliştirme, Test gibi ortam veri
tabanlarının canlı ortam veri tabanı ile aynı sunucuda olması çeşitli
sorunlara yol açabilir. PaperWork bunu tavsiye etmez. Üretim ortamının
diğer ortamlardan yalıtılmış olması tavsiye edilir.

PaperWork veri tabanı kapalı bir kutu gibi çalışır. Bu nedenle sistem
tabloları üzerinde DML (insert/delete/update) işlemleri yapılmamalıdır.
Kayıtlarda yapılmasına ihtiyaç duyulan her türlü işlem geliştirme
katmanı aracılığı ile yapılmalıdır.

:::::::::::: {.section .infoBox}
::: title
Sistem Veri Tabanı Bakımı
:::

:::::::::: content
::: {style="text-align: justify;"}
PaperWork mimarisinde sistem tanımlı işleri bulunur. Bu işlerin bazıları
1 dakikada 1, bazıları 1 saatte 1, bazıları her gün sonunda, sonuncusu
ise her hafta cumartesi gecesi saat 01:00 da çalışır. Çalışan bu tanımlı
işler sistem bakımını otomatik yapar.
:::

::: {style="text-align: justify;"}
Pazar günü 01:00 da çalışan tanımlı iş, veri tabanındaki tüm indeksleri
düzenler, istatistikler günceller. Veri tabanının olması gerektiği gibi
çalışmasını sağlar. Bunun yanında kurumun sadece yedekleme sürecini
planlaması yeterlidir.
:::

\

::: {style="text-align: justify;"}
**Not: **GGSoft, veri tabanı desteği vermez. İleri seviyede veri tabanı
işlemleri için üretici ile irtibata geçilmelidir.
:::

::: {style="text-align: justify;"}
**Not: **Bu metod istenirse veri tabanı üzerindeki
**PR_MAINT_DB **prosedürü tetiklenerek işlem yapılabilir.
:::

::: {style="text-align: justify;"}
**Not: **Sistem parametreleri tablosuna (PW_SYSTEM_PARAMS)
MAINTANANCE_PERIOD isimli parametre eklenerek bu periyot
değiştirilebilir. Bu parametre tanımlı değil ise değeri 7\'dir ve 7
günde 1 çalışır. Bu değerin 2 yapılması demek ilgili metodun 2 günde 1
çalışmasını sağlar.
:::

::: {style="text-align: justify;"}
**Not: **Verinin çok olduğu durumlarda bu işlem çok uzun sürebilir ve
işlem mesai saatlerine sarkabilir. MS SQL Standart versiyonu bu işlemi
yaparken tablolar kilitlenir ve başka işlem yapılmasına izin vermez. MS
SQL Server Enterprise versiyonu işlemler devam ederken indexlerin
düzenlenmesine izin verir. Böyle bir durum yaşanıyor ise **MS SQL Server
2019 Enterprise** ve üzeri bir versiyon kullanılmalıdır.
:::

::: {style="text-align: justify;"}
**Not: **WorkFlow veri tabanının bakımı her gece otomatik olarak
yapılır. Eğer bakım yapılmasın isteniyor ise MAINT_WF_DB parametresi
parametreler ekranından değiştirilmelidir.
:::

:::::::: {.section .infoBox}
::: title
MS SQL Shrink
:::

:::::: content
::: {style="text-align: justify;"}
Zaman zaman müşterilerimizden MS SQL Server log dosyası hakkında destek
talebi gelmektedir. Bu dosyaların shrink edilmesi talep edilmektedir.
:::

::: {style="text-align: justify;"}
1\. PaperWork destek ekibi veri tabanı bakımı konusunda destek vermez.
:::

::: {style="text-align: justify;"}
2\. Microsoft da dahil MS SQL Server sistem yöneticileri, veri tabanı
**shrink **işlemini uygun görmezler. [Şu
adresten](https://www.brentozar.com/archive/2009/08/stop-shrinking-your-database-files-seriously-now/){rel="nofollow noopener noreferrer"
target="_blank" translate="no"} bu konudaki makaleye erişebilirsiniz.
Eğer dilerseniz siz bu işlemi gerçekleştirebilirsiniz.
:::

:::::: {.section .errorBox}
::: title
Lisans
:::

:::: content
::: {style="text-align: justify;"}
Microsoft SQL Server birden fazla lisanslama tipine sahiptir. CAL
lisansı ile kullanıcı başına, Core lisansı ile de sunucu CPU başına
lisanslama opsiyonları bulunur. PaperWork kullanıcıları direk veri
tabanına erişmedikleri için CAL lisansı uygun [değildir]{.underline}.
Bununla beraber Lisans sorumluluğu sunucu işletim sistemlerinde olduğu
gibi alıcıdadır.
